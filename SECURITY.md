# 安全政策

## 支持的版本

| 版本 | 支持状态 |
| ---- | ------- |
| 1.0.x | ✅ 积极维护 |

## 报告漏洞

如果您发现安全漏洞，请**不要**创建公开的 Issue。

### 报告方式

请通过以下方式私下报告：

1. 发送邮件至项目维护者（如适用）
2. 使用 GitHub 的私密漏洞报告功能

### 报告内容

请包含以下信息：

- 漏洞类型（如 XSS、SQL 注入、认证绕过等）
- 受影响的组件/文件路径
- 详细的复现步骤
- 潜在影响评估
- 可能的修复建议（如有）

### 响应时间

- 我们会在 **48 小时** 内确认收到报告
- 我们会在 **7 天** 内提供初步评估
- 根据严重程度，修复将在 **30 天** 内发布

### 处理流程

1. 收到报告后进行验证
2. 评估漏洞严重程度
3. 开发并测试修复方案
4. 发布安全更新
5. 公开致谢报告者（如报告者同意）

## 最佳实践

部署本项目时，请确保：

- [ ] 修改默认的 JWT_SECRET 为强随机密钥
- [ ] 在生产环境中使用 HTTPS
- [ ] 正确配置 CORS_ORIGIN
- [ ] 定期更新依赖包
- [ ] 备份数据库
- [ ] 监控异常登录

感谢您帮助保护 AI Hub 的安全！
